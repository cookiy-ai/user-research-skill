# Thematic Analysis

Thematic analysis is a reliable default for turning messy qualitative inputs into findings.

## Working Sequence

1. Familiarize
   - Read or skim all inputs once before naming themes.
2. Capture observations
   - Write down single observations, behaviors, quotes, or frictions.
3. Code
   - Label observations with short descriptive tags.
4. Cluster
   - Group related codes into candidate themes.
5. Refine
   - Split clusters that hide multiple ideas.
   - Merge clusters that are saying the same thing.
6. Write findings
   - Convert clusters into clear findings supported by evidence.

## What Counts As Good Evidence

- repeated behavior across participants
- strong workarounds
- decision criteria that show up in real examples
- contradictions that reveal segments
- analytics or support signals that reinforce the qualitative theme

## What Does Not Count As A Finding

- one memorable quote with no pattern behind it
- a product idea from a participant
- a restatement of a survey question
- the researcher's prior belief

## Confidence Heuristic

- High: supported by multiple participants or multiple source types
- Medium: repeated signal but narrow sample or incomplete triangulation
- Low: early signal, directional only, or weak support
